1) ¿Por qué las clases que utilizan templates se declaran y definen en los .h?

Los templates en C++ se utilizan y se declaran en los archivos .h porque el compilador necesita tener acceso a la definicion completa de la clase o función template en el momento de la compilación. Esto se debe a que los templates son instanciados en tiempo de compilación, y el compilador necesita conocer todos los detalles de la implementación para generar el código adecuado para cada tipo específico que se utilice con el template. 

2) ¿Qué es un functor? Ejemplifique.

Un functor es un objeto que se comporta como una funcion, es decir, puede ser llamada como si fuera una funcion. Esto se logra sobrecargando el operador de llamada a función `operator()`. Los functors son útiles para pasar comportamientos como argumentos a funciones o algoritmos y pueden mantener estado interno, a diferencia de las funciones normales.
Ejemplo de un functor:
```C++

#include <iostream>
class Producto {
private:
    int factor_;
public:
    Producto(int factor) : factor_(factor) {}
    
    int operator()(int x) const {
        return x * factor_;
    }
}
int main() {
    Producto multiplicador(5);
    std::cout << "Resultado: " << multiplicador(10) << std::endl; // Imprime 50
    return 0;
}


```

b- `static int *C[3];` esta es una declaracion de un puntero a un arreglo de 3 punteros a enteros, static significa que la variable tiene alcance global si esta por fuera de una funcion o local si esta por dentro, y su valor se mantiene entre llamadas a la funcion.
c- `static short F(const float *a);` Es una declaracion de una funcion que recibe un puntero a un float constante y devuevle un short, static significa que la funcion tiene alcance local al archivo donde se define y no puede ser llamada desde otros archivos.

4) ¿Qué es un Deadlock? Ejemplifique.
   Un deadlock es una situacion donde uno o mas procesos quedan bloqueados esperando que un recurso que esta siendo utilizado por otro proceso, sea desbloqueado y este otro proceso a su vez esta esperando por un recurso que el primer proceso tiene, generando un ciclo de espera que impide que los procesos puedan continuar su ejecucion.
Ejemplo de deadlock:
```C++
#include <iostream>
#include <thread>
#include <mutex>
std::mutex m1, m2;

void thread1() {
    m1.lock();
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    m2.lock();
    // Código crítico
    m2.unlock();
    m1.unlock();
}
void thread2() {
    m2.lock();
    std::this_thread::sleep_for(std::chrono::milliseconds(100));
    m1.lock();
    // Código crítico
    m1.unlock();
    m2.unlock();
}
int main() {
    std::thread t1(thread1);
    std::thread t2(thread2);
    t1.join();
    t2.join();
    return 0;
}

```

6) ¿Qué significa que una función es blocante?¿Cómo subsanaría esa limitación en término de
mantener el programa ‘vivo’ ?

Una funcion significa que es bloqueante cuando su ejecucion esta bloqueando el flujo del programa actual, no permitiendo que otras operaciones continuen con sus tareas hasta que termine este proceso. Una solucion para que el programa continue vivo seria la utilizacion de hilos perimitiendo que otras operaciones se ejecuten en paralelo mientras la funcion bloqueante esta en proceso.

7) Explique qué es y para qué sirve un constructor MOVE en C++. Indique cómo se comporta el
sistema si éste no es definido por el desarrollador.

    Un constructor MOVE es un constructor que permite transferir todos los recursos del objeto fuente al objeto destino, en lugar de copiar los recursos. Deja en estado valido al objeto fuente, pero sin los recursos que fueron movidos. Esto es útil para optimizar el rendimiento y evitar copias innecesarias de objetos grandes o costosos de copiar. Si el desarrollador no define un constructor MOVE, el compilador genera uno por defecto que puede llevar a problemas de rendimiento por malas copias

```C++
class MiClase {
    private:
        int data;
    public:
        Miclase(int d) : data(d) {}
        // Constructor MOVE
        Miclase(Miclase&& other) noexcept : data(other.data) {
            other.data = 0; // Deja el objeto fuente en un estado válido
        }
        // Constructor de copia
        Miclase(Miclase& other) : data(other.data) {}
};
```

8) Escribir un programa ISO C que procese el archivo “valuesword.dat” sobre sí mismo,
eliminando los words (2 bytes) múltiplos de 16.

```C
int main() {
    FILE *f=fopen("valuesword.dat", "r+b");
    if(f==NULL) {
        perror("Error al abrir el archivo");
        return 1;
    }
    int read, write;
    read = write = 0;
    short value;
    while(1){
        fseek(f, read, SEEK_SET);
        fread(&value, sizeof(short), 1, f);
        if(feof(f)) break;
        if(value % 16 != 0) {
            fseek(f,write, SEEK_SET);
            fwrite(&value, sizeof(short), 1, f);
            write += sizeof(short);
        }
        read += sizeof(short);
    }
    fclose(f);
    return 0;
}
```

9) Implemente la función void ValorHex(char *hex, int *ent) que interprete la cadena hex (de
símbolos hexadecimales) y guarde el valor correspondiente en el entero indicado por ent.

```C++
void ValorHex(char *hex, int *ent) {
    *ent = 0; // Inicializa el entero a 0
    while (*hex) {
        char c = *hex++;
        int value;
        if (c >= '0' && c <= '9') {
            value = c - '0';
        } else if (c >= 'A' && c <= 'F') {
            value = c - 'A' + 10;
        } else if (c >= 'a' && c <= 'f') {
            value = c - 'a' + 10;
        } else {
            // Manejo de error para caracteres no válidos
            return;
        }
        *ent = (*ent << 4) | value; // Desplaza a la izquierda y agrega el nuevo valor
    }
}

```


10)  Implemente una función C++ denominada Interseccion que reciba dos listas de elementos y
devuelva una nueva lista con los elementos que se encuentran en ambas listas:
std::list<T> Interseccion(std::list<T> a,std::list<T> b);

```C++
template<typename T>
std::list<T> Interseccion(std::list<T> a, std::list<T> b) {
    std::list<T> resultado;
    for(const auto& elemento : a) {
        if (std::find(b.begin(), b.end(), elemento) != b.end()) {
            resultado.push_back(elemento);
        }
    }
    return resultado;
}
```

1) ¿Qué es un functor? Ejemplifique.

Un functor es un objeto que puede ser utilizado como si fuera una funcion. La ventaja de los functors es que pueden mantener el estado interno, a diferencia de las funciones normales. Esto se logra sobrecargando el operador de llamada a funcion `operator()`. Los functors son útiles para pasar comportamientos como argumentos a funciones o algoritmos.
Ejemplo de un functor:
```C++
#include <iostream>
class Producto {
    private:
        int x;
    public:
        Producto(int factor) : x(factor) {}
        int operator()(int y) const {
            return x * y;
        }
};
int main() {
    Producto multiplicador(5);
    std::cout << "Resultado: " << multiplicador(10) << std::endl; // Imprime 50
    return 0;
}
```

2) Explique qué es cada uno de los siguientes, haciendo referencia a su inicialización, su
comportamiento y el area de memoria donde residen:
a) Una variable global static
b) Una variable local
c) Un atributo de clase static.

a- Una variable global static es una variable que se declara por fuera de cualquier tipo de funcion y se puede utilizar dentro de todo el archivo en el cual esta declarado, pero solo en ese archivo ya que el static hace que su scope sea limitado. Su inicializacion se realiza al principio del programa y reside en el data segment de la memoria.
b- una variable local es una variable que se declara dentro de una funcion y su inicializacion se realiza cada vez que la funcion es llamada, no se conserva su valor entre llamadas y reside en el stack de la memoria.
c- Un atributo de clase static es una variable que pertenece a la clase en si y puede ser utilizado unicamente por la clase y por sus instancias. Su inicializacion se realiza al principio del programa y reside en el data segment de la memoria si esta inicializada o en el BSS si no.

4) Escriba un programa que imprima por salida estándar los números entre 1 y 100, en orden
ascendente. Se pide que los números sean contabilizados por una variable global única y que
los pares sean escritos por un hilo mientras que los impares sean escritos por otro.
Contemple la correcta sincronización entre hilos y la liberación de los recursos utilizados.

```C++
#include <iostream>
#include <thread>
#include <mutex>
#include <condition_variable>
std::mutex mtx;
std::condition_variable cv;
static int contador = 1;
void imprimirPares(){
    while(true){
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock, []{ return contador <= 100 && contador % 2 == 0; });
        if(contador > 100) {
            cv.notify_all();
            return;
        }
        std::cout << contador << std::endl;
        ++contador;
        cv.notify_all();
    }
}
void imprimirImpares(){
    while(true){
        std::unique_lock<std::mutex>lock(mtx);
        cv.wait(lock, []{ return contador <= 100 && contador % 2 != 0; });
        if(contador > 100) {
            cv.notify_all();
            return;
        }
        std::cout << contador << std::endl;
        ++contador;
        cv.notify_all();
    }
}

int main(){
    std::thread t1(imprimirPares);
    std::thread t2(imprimirImpares);
    t1.join();
    t2.join();
    return 0;
}


```

5) ¿Qué es la compilación condicional? En qué etapa del proceso de transformación de código se resuelve. Ejemplifique mediante código C, dando un caso de uso útil.

La compilacion condicional es una tecnica que permite que no se ejecuten partes del codigo fuente que no son necesarias en el compilado deseado. Esto se resuelve en la etapa de preprocesamiento del codigo fuente, antes de la compilacion. En la cual tambien se expanden las macros y se incluyen los archivos de cabecera.
ejemplo de compilacion condicional:
```C
#include <stdio.h>
#define DEBUG 1
int main() {
    int x = 5;
    #if DEBUG
        printf("Debug: x = %d\n", x);
    #endif
    return 0;
}
```

8) ¿Qué significa la palabra virtual antepuesta a un método de una clase?¿Qué cambios genera a nivel compilación y al momento de ejecución?

La palabra virtual antepuesta a un metodo de una clase indica que el metodo puede ser modificado por una clase derivada. Esto permite que se pueda utilizar el polimorfismo en tiempo de ejecucion. Cuando se ejecuta el programa el compilador genera una tabla de funciones virtuales que contienen punteros a las funciones virtuales de la clase.

```C++
class Figura {
    public:
        virtual float area() const = 0; // metodo virtual puro
}       ~Figura() {}

class cuadrado : public Figura {
    private:
        float lado;
    public:
        cuadrado(float l) : lado(l) {}
        float area() const override {
            return lado * lado;
        }
        ~cuadrado() {}
}
class circulo : public Figura {
    private:
        float radio;
    public:
        circulo(float r) : radio(r) {}
        float area() const override {
            return 3.14 * radio * radio;
        }
}
int main(){
    Circulo c(5);
    Cuadrado q(4);
    std::cout << "Area del circulo: " << c.area() << std::endl;
    std::cout << "Area del cuadrado: " << q.area() << std::endl;
    return 0; 

}
```

9) Escriba un programa (desde la inicialización hasta la liberación de los recursos) que reciba paquetes de texto delimitados por corchetes angulares (“<...>”) y los imprima completos por pantalla. Al recibir un paquete vacío (“<>”) debe cerrarse ordenadamente. No considere errores.

```C++
#include <iostream>
#include <string>
#include <socket>
int main() {
    struct addrinfo hints;
    struct addrinfo *res;
    memset(&hints, 0, sizeof(hints));
    hints.ai_family = AF_INET;
    hints.ai_socktype = SOCK_STREAM;
    getaddrinfo("localhost", "8080", &hints, &res);
    int server_fd = socket(res->ai_family, res->ai_socktype, res->ai_protocol);
    bind(server_fd, res->ai_addr, res->ai_addrlen);
    listen(server_fd, 1);
    int client_fd = accept(server_fd, NULL, NULL);
    std::string paquete;
    while(true){
        char c;
        c = recvall(client_fd, &c, 1);
        if(c =='<'){
            paquete.clear();
        } else if(c == '>')
        {
            if(paquete == "<>")
                break;
            std::cout << paquete << std::endl;
        } else {
            paquete += c;
        }
    }
    close(client_fd);
    close(server_fd);
    freeaddrinfo(res);
    return 0;

}

```

10)   Describa el concepto de loop de eventos (events loop) utilizado en programación orientada a eventos y, en particular, en entornos de interfaz gráfica (GUIs).

    Un loop de eventos es un patron de programacion utilizado principalmente en entornos de interfaz grafica el cual se basa en un bucle(normalmente una cola) que espera constantemente a que ingresen eventos para ir procesandolos uno a uno. Los eventos pueden ser generados por el usuario o por el sistema operativo. El loop de eventos permite que la aplicacion permanezca en ejecucion y responda a las acciones del usuario sin bloquear la interfaz grafica, ya que los eventos se procesan de manera asincronica.

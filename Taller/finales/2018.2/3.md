1) Explique qué es y para qué sirve un constructor de copia en C++. 
    a) Indique cómo se comporta el sistema si éste no es definido por el desarrollador; 
    b) Explique al menos una estrategia para evitar que una clase particular sea copiable;
    c) Indique qué diferencia existe entre un constructor de copia y uno move.

    Un constructor en copia sirve para crear un nuevo objeto con los mismos valores que otro objeto ya existente. Es útil para pasar objetos por valor, retornar objetos desde funciones y para inicializar un objeto con otro.
    a- Si el constructor por copia no esta definido por el desarrollador el compilador provee un constructor por defecto que realiza una copia superficial de los miembros del objeto. Esto puede causar problemas de punteros o recursos compartidos.
    b- Una estategia para evitar que sea copiable es declarar el constructor de copia como privado o eliminarlo (C++11) usando `= delete`.
    c- La diferencia entre un constructor en copia y un move es que el constructor en copia crea una copia del objeto mientras que el move transfiere la propiedad de los recursos del objeto original al nuevo objeto, dejando el objeto original en un estado válido pero indefinido.

una cantidad (Q). La función debe retornar una copia de S salvo los caracteres que se
encuentran entre los índices I1 e I2 que serán duplicados Q veces.
Ej. replicar(“Hola”, 1, 2, 3) retorna “Hololola”.

```C
char *replicar(char *S, int I1, int I2, int Q)
{
    int len = strlen(S);
    int new_len = len + (I2 - I1 + 1) * Q
    char *new_str = malloc(new_len + 1);
    for (int i = 0; i < I1; i++)
    {
        new_str[i] = S[i];
    }
    for (int i= 0; i<Q; i++){
        for (int j= I1; j<= I2; j++){
            new_str[I1 + i*(I2 - I1 + 1) + (j - I1)] = S[j];
        }
    }
    if (I2 < len - 1)
    {
        for (int i = I2 + 1; i < len; i++)
        {
            new_str[I1 + Q * (I2 - I1 + 1) + (i - I2 - 1)] = S[i];
        }
    }
    new_str[new_len] = '\0';
    return new_str; 
}
```

4) Escribir un programa ISO C que reciba por argumento el nombre de un archivo de texto y lo
procese sobre sí mismo (sin crear archivos intermedios ni subiendo todo su contenido a
memoria). El procesamiento consiste en eliminar las líneas de 1 sola palabra.

```C
#define MAX_LINE_LENGTH 1024

int main(int argc, char *argv[]){
    FILE *f= fopen(argv[1], "r+");
    if (f == NULL){
        perror("Error al abrir el archivo");
        return 1;
    }
    int rpos=0, wpos=0;
    char line[MAX_LINE_LENGTH];
    while(1){
        char c = fgetc(f);
        if (c == EOF){
            break;
        }
        if (c == '\n'){
            line[rpos] = '\0';
            if (strchr(line, ' ') != NULL){
                fseek(f, wpos, SEEK_SET);
                fputs(line, f);
                fputc('\n', f);
                wpos = ftell(f);
            } 
        }
        fseek(f, rpos, SEEK_SET);
        rpos++;
    }
    close(f);
    return 0;
}

```



5) Declare una clase de elección libre. Incluya todos los campos de datos requeridos con su
correcta exposición/publicación, y los operadores ++, -, ==, >> (carga), << (impresión),
constructor move y operador float().

```C++
class MiClase {
    private:
        int valor;
    public:
        MiClase(int v) : valor(v) {}
        MiClase(MiClase&& other)noexcept : valor(other.valor) {
            other.valor = 0;
        }
        MiClase& operator++() {
            ++valor;
            return *this;
        }
        MiClase operator-(const MiClase& other) const {
            return MiClase(valor - other.valor);
        }
        bool operator==(const MiClase& other) const {
            return valor == other.valor;
        }
        friend std::istream& operator>>(std::istream& is, MiClase& obj) {
            is >> obj.valor;
            return is;
        }
        friend std::ostream& operator<<(std::ostream& os, const MiClase& obj) {
            os << obj.valor;
            return os;
        }
        operator float() const {
            return static_cast<float>(valor);
        }
}
```

en cada paquete. Un paquete está definido como una sucesión de números recibidos como
texto, en decimal, separados por comas y terminado con un signo igual (ej: “27,12,32=”). Al
recibir el texto ‘FIN’ debe finalizar el programa ordenadamente liberando los recursos.

```C++
int main(int argc, char *argv[]){
    int puerto = atoi(argv[1]);
    const char* ip = argv[2];
    struct addr_in hints;
    struct addr_in *res;
    memset(&hints, 0, sizeof(hints));
    hints.ai_family = AF_INET;
    hints.ai_socktype = SOCK_STREAM;
    hints.ai_flags = AI_PASSIVE;
    getaddrinfo(ip, puerto, &hints, &res);

    int server_fd = socket(res-> ai_family, res->ai_socktype, res->ai_protocol);
    bind(server_fd, res->ai_addr, res->ai_addrlen);
    listen(server_fd, 1);
    int client_fd = accept(server_fd, NULL, NULL);
    int num = 0;
    int sum = 0;
    std::string buffer;
    
    while(true){
        char c;
        recvall(client_fd, &c, 1, 0);
        buffer += c;

            break;

            sum += num;
            num = 0;
            buffer.clear();
            std::cout << "Sumatoria: " << sum << std::endl;

            sum += num;
```

9)  ¿Qué ventaja ofrece un lock raii frente al tradicional lock/unlock?

Un lock raii es una tecnica de manejo de recursos que consiste en la liberacion de recursos de manera automatica cuando el no se necesitan mas o salen del scope como por ejemplo archivos o locks. La ventaja frente al lock tradicional es que libera la responsabilidad del programador de liberar el recurso, evitando errores. Tambien permiterefactorizacion a futuro mas simple y segura. Al mismo tiempo permite un manejo mas seguro de excepciones, ya que si ocurre una excepcion el lock raii se encargara de liberar el recurso automaticamente.


10)  ¿Qué significa que una función es blocante?¿Cómo subsanaría esa limitación en términos de mantener el programa ‘vivo’ ?

    Una funcion bloqueante es cuando el proceso se ve afectado por esa funcion no permitiendo que el programa continue su correcto funcionamiento hasta que termine de ejecutarse. Una forma de subsanar y mantener el programa vivo seria mediante el uso de hilos para que la funcion bloqueante se ejecute en un hilo separado y el programa principal pueda continuar su ejecucion sin verse afectado.

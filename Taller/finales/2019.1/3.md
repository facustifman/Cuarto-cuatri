2) ¿Qué es un functor? ¿Qué ventaja ofrece frente a una función convencional? Ejemplifique.
Un functor es un objeto que se puede usar como si fuera una función. En C++ se logra definiendo una clase o struct que sobrecarga el operador operator().
La ventaja frente a una función convencional es que el functor puede guardar estado interno. Es decir, además de ejecutar una operación, puede recordar datos propios entre llamadas.
```C++

#include <iostream>

class Sumador {
private:
    int base;

public:
    Sumador(int base) : base(base) {}

    int operator()(int x) {
        return base + x;
    }
};

int main() {
    Sumador sumar5(5);

    std::cout << sumar5(10) << std::endl; // imprime 15
    std::cout << sumar5(3) << std::endl;  // imprime 8

    return 0;
}
```


3) Escriba un programa que imprima por salida estándar los números entre 1 y 100, en orden ascendente. Se pide que los números sean contabilizados por una variable global única y que los pares sean escritos por un hilo mientras que los impares sean escritos por otro.
Contemple la correcta sincronización entre hilos y la liberación de los recursos utilizados.

```C++
#include <threads>
#include <mutex>
#include <condition_variable>

static int numero = 1;
std ::mutex mtx;
std ::condition_variable cv;

void imprimir_impares(){
    while (true){
        std::unique_lock<std::mutex> lock(mtx);
        cv.wait(lock,[]{
            return numero>100 || numero%2 !=0;
        });
        if (numero > 100)[
            cv.notify_all();
            return;
        ]
        std:: cout<< numero << std::endl;
        numero++;
        cv.notify_all()
    }
}

void imprimir_pares(){
    while(true){
        std::unique_lock<std::mutex>lock(mtx);

        cv.wait(lock,[]{
            return numero>100 || numero%2 == 0
        });

        if (numero>100){
            cv.notify_all();
            return;
        }

        std::cout << numero << std::endl;
        numero++;
        cv.notify_all();

    }
}

int main(){
    std:: thread t_impares(imprimir_impares);
    std:: thread t_pares(imprimir_pares);

    t_impares.join();
    t_pares.join();

    return 0;
}
```

6) ¿Por qué las librerías que usan Templates se publican con todo el código fuente y no como
un .h y .o/.obj?.

Porque los templates en C++ se instancian en tiempo de compilación.
Cuando el compilador encuentra algo como:
std::vector<int> v;
necesita ver el código completo del template para generar una versión concreta de esa clase para int. Lo mismo pasaría con vector<float>, vector<Persona>, etc. Cada tipo usado puede requerir una instanciación distinta.
Por eso no alcanza con tener solo un .h con declaraciones y un .o/.obj ya compilado: el archivo objeto no puede contener todas las combinaciones posibles de tipos que el usuario podría usar.
Entonces, las librerías con templates suelen publicarse con el código completo en headers, para que el compilador pueda generar las instancias necesarias en cada programa.


7) ¿Qué características debe tener un compilador C para se considerado “portable”?
Un compilador C es considerado portable si permite compilar programas en distintas plataformas manteniendo el mismo comportamiento definido por el estándar.
Debe cumplir principalmente con estas características:
Respetar el estándar del lenguaje C, por ejemplo ANSI C / ISO C.
- No depender de una arquitectura específica, sistema operativo o procesador particular.
- Definir correctamente los tipos básicos según la plataforma, pero respetando las reglas del estándar.
Proveer las bibliotecas estándar de C, como stdio.h, stdlib.h, string.h, etc.
- Generar código equivalente para distintas máquinas sin exigir cambios grandes en el código fuente.
- Evitar extensiones propias obligatorias del compilador para que el programa funcione.
- Manejar de forma consistente el preprocesador, las directivas #include, #define, compilación condicional, etc.
En resumen: un compilador C portable debe ajustarse al estándar y permitir que un programa escrito en C estándar pueda compilarse en diferentes entornos con pocas o ninguna modificación.

10)   ¿Qué diferencia existe entre un constructor por copia y uno por movimiento?
Ejemplifique.

Un constructor por copia crea un objeto nuevo copiando el contenido de otro objeto existente. El objeto original queda igual.
Un constructor por movimiento crea un objeto nuevo tomando los recursos de otro objeto temporal o descartable. En vez de duplicar memoria, archivos, sockets, etc., “roba” esos recursos y deja al objeto original en un estado válido pero vacío

```C++
class miclase{
    private:
        int x
    public:

        miclase(int i) : x(i) {}

        miclase(const miclase& i) : x(i.x) {}
        
        miclase(miclase&& i) noexcept : x(i.x){ i. x = 0;}
        
}

```

1 - Explique qué es y para qué sirve un constructor de copia en C++. a) Indique cómo se
comporta el sistema si éste no es definido por el desarrollador; b) Explique al menos una
estrategia para evitar que una clase particular sea copiable; c) Indique qué diferencia existe
entre un constructor de copia y uno move.

Un constructor de copia en C++ es un tipo especial de constructor que se utiliza para crear una nueva instancia de una clase a partir de otra instancia existente. Su propósito principal es duplicar el estado de un objeto, asegurando que los recursos dinámicos (como memoria asignada en el heap) sean correctamente gestionados para evitar problemas como la doble liberación de memoria.

a) Si un constructor de copia no es definido por el desarrollador, el compilador proporciona uno por defecto. Este constructor de copia realiza una copia bit a bit de los miembros de la clase, lo que puede causar problemas si la clase contiene punteros o recursos dinámicos.

b) Una estrategia para evitar que una clase sea copiable es declarar el constructor de copia y el operador de asignación como privados y no implementarlos. Esto impide que otros códigos intenten crear copias de objetos de esa clase.

c) La diferencia entre un constructor de copia y un constructor move es que el primero crea una nueva instancia a partir de otra existente, mientras que el segundo traslada los recursos de un objeto a otro, dejando al objeto original en un estado válido pero indefinido.

2)Escriba una función ISO C llamada Replicar que reciba 1 cadena (S), dos índices (I1 e I2) y
una cantidad (Q). La función debe retornar una copia de S salvo los caracteres que se
encuentran entre los índices I1 e I2 que serán duplicados Q veces.
Ej. replicar(“Hola”, 1, 2, 3) retorna “Hololola”.

Aquí tienes una implementación en C de la función `Replicar` que cumple con los requisitos especificados:

```c
#include <stdio.h>
#include <stdlib.h>

char* Replicar(const char* S, int I1, int I2, int Q) {
    int len = strlen(S);
    if (I1 < 0 || I2 >= len || I1 > I2) {
        return NULL; // Índices inválidos
    }
    char* result = (char*)malloc(len + (I2 - I1 + 1) * Q + 1); // +1 para el terminador nulo
    if (!result) {
        return NULL; // Error de asignación de memoria
    }
    for (int i = 0; i < I1; i++) {
        result[i] = S[i]; // Copiar la parte antes de I1
    }
    for (int j = 0; j < Q; j++) {
        for (int k = I1; k <= I2; k++) {
            result[I1 + j * (I2 - I1 + 1) + (k - I1)] = S[k]; // Duplicar los caracteres entre I1 e I2
        }
    }
    if (I2 + 1 < len) {
        for (int i = I2 + 1; i < len; i++) {
            result[len + (I2 - I1 + 1) * Q - (len - i)] = S[i]; // Copiar la parte después de I2
        }
    }

    result[len + (I2 - I1 + 1) * Q] = '\0'; // Agregar el terminador nulo
    return result;
}
```

4)Escribir un programa ISO C que reciba por argumento el nombre de un archivo de texto y lo
procese sobre sí mismo (sin crear archivos intermedios ni subiendo todo su contenido a
memoria). El procesamiento consiste en eliminar las líneas de 1 sola palabra.

```C
#include <stdio.h>
#include <string.h>
#include <unistd.h>

int main(int argc, char *argv[]) {
    if (argc != 2) {
        return 1;
    }
    FILE *f = fopen(argv[1], "r+");
    if (f == NULL) {
        perror("Error al abrir el archivo");
        return 1;
    }
    char linea[1024];
    long writePos = 0;

    while (fgets(linea, sizeof(linea), f)) {
        long readPos = ftell(f); // Guarda la posi de lectura actual

        int esUnaPalabra = (linea[0] != '\n') && (strchr(linea, ' ') == NULL);

        if (!esUnaPalabra) {
            fseek(f, writePos, SEEK_SET); // Mueve el puntero de escritura a la posición correcta
            fwriteall(linea, 1, strlen(linea), f);
            writePos += strlen(linea);
        }

        fseek(f, readPos, SEEK_SET);// Restaurar la posición de lectura, seek set desde el principio del archivo
    }

    ftruncate(fileno(f), writePos);
    fclose(f);
    return 0;
}
```

5)Declare una clase de elección libre. Incluya todos los campos de datos requeridos con su correcta exposición/publicación, y los operadores ++, -, ==, >> (carga), << (impresión), constructor move y operador float().

```C++
#include <iostream>

class MiClase {
private:
    int valor;
public:
    // Constructor
    MiClase(int v) : valor(v) {}
    int getValor() const { return valor; }
    MiClase& suma(){
        ++valor;
        return *this;
    }
    MiClase& resta(){
        --valor;
        return *this;
    }
    bool igual(const MiClase& other) const {
        return valor == other.valor;
    }
    friend std::ostream& ingresar(std::ostream& os, const MiClase& obj) {
        os << obj.valor;
        return os;
    } // se usa friend para permitir el acceso a los miembros privados de la clase desde la función de sobrecarga del operador de inserción.
    friend std::istream& cargar(std::istream& is, MiClase& obj) {
        is >> obj.valor;
        return is;
    }
    // Constructor move
    MiClase(MiClase&& other) noexcept : valor(other.valor) {
        other.valor = 0; // Dejar el objeto original en un estado válido
    }
    float toFloat() const {
        return static_cast<float>(valor);
    }
};
```

6 )¿Qué es una macro de C? Describa las buenas prácticas para su definición y ejemplifique.

Una macro de C es una directiva del procesador que permite definir fragmentos de código que pueden ser reutilizados en diferentes partes del programa. Se definen utilizando la palabra clave `#define` y pueden ser utilizadas para crear constantes, funciones en línea o para realizar operaciones condicionales.
Buenas prácticas para la definición de macros incluyen:

- Usar nombres en mayúsculas para las macros, para diferenciarlas de las variables y funciones.
- Evitar el uso de macros para funciones complejas; en su lugar, utilizar funciones en línea o plantillas.
- Documentar claramente el propósito de la macro.

Ejemplo de macro:

```c
#define PI 3.14159
#define MAX(a, b) ((a) > (b) ? (a) : (b))
```

7)Escriba un programa que reciba por línea de comandos un Puerto y una IP. El programa
debe aceptar una única conexión e imprimir en stdout la sumatoria de los enteros recibidos
en cada paquete. Un paquete está definido como una sucesión de números recibidos como
texto, en decimal, separados por comas y terminado con un signo igual (ej: “27,12,32=”). Al
recibir el texto ‘FIN’ debe finalizar el programa ordenadamente liberando los recursos.

```C++
#include <iostream>
#include <cstdlib>
#include <cctype>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

int main(int argc, char *argv[]) {
    if (argc != 3) {
        return 1;
    }

    const char *ip = argv[1];
    int puerto = atoi(argv[2]);

    int sockfd = socket(AF_INET, SOCK_STREAM, 0);

    sockaddr_in addr{};
    addr.sin_family = AF_INET;
    addr.sin_port = htons(puerto);
    inet_pton(AF_INET, ip, &addr.sin_addr);

    if(bind(sockfd, (struct sockaddr*)&addr, sizeof(addr))){
        return 1;
    };
    listen(sockfd, 1);

    int client_fd = accept(sockfd, nullptr, nullptr);

    char buffer[1024];
    int n;
    int numero = 0;
    int suma = 0;
    int finMatched = 0;
    const char *FIN = "FIN";

    while ( true ) {
        n = recvall(client_fd, buffer, sizeof(buffer));
        for (int i = 0; i < n; i++) {
            char c = buffer[i];

            if (isdigit(c)) {
                numero = numero * 10 + (c - '0');
                finMatched = 0;
            } else if (c == ',') {
                suma += numero;
                numero = 0;
                finMatched = 0;
            } else if (c == '=') {
                suma += numero;
                std::cout << suma << std::endl;
                suma = 0;
                numero = 0;
                finMatched = 0;
            } else if (c == FIN[finMatched]) {
                finMatched++;
                if (finMatched == 3) {
                    close(client_fd);
                    close(sockfd);
                    return 0;
                }
            } else {
                finMatched = (c == FIN[0]) ? 1 : 0;
            }
        }
    }

    close(client_fd);
    close(sockfd);
    return 0;
}
```

8 )Describa el proceso de transformación de código fuente a un ejecutable. Precise las etapas, las tareas desarrolladas y los tipos de error generados en cada una de ellas.

Proceso de transformación de código fuente a un ejecutable:

1. Preprocesamiento: En esta etapa, el procesador de C/C++ realiza todas las directivas de preprocesamiento, como la inclusion de archivos de encabezado, la expansion de macros y la eliminacion de comentarios. Los errores comunes en esta etapa incluyen errores de sintaxis en las directivas de preprocesamiento y problemas con la inclusión de archivos.
2. Compilacion: El compilador traduce el codigo fuente a codigo objeto, que es un formato intermedio. Durante esta etapa, se realizan verificaciones de sintaxis y semántica. Los errores comunes incluyen errores de sintaxis, tipos de datos incompatibles y referencias a funciones o variables no definidas.
3. Enlazado: El linker combina los archivos de codigo objeto y las biblotecas necesarias para crear un ejecutable. En esta etapa, se resuelven las referencias a funciones y variables externas. Los errores comunes que incluye es que las referencias a funciones o variables no definidas, conflictos de nombres y problemas con las bibliotecas compartidas.
4. Generacion del ejecutable :  En esta etapa final, el linker genera el archivo ejecutable final con el codigo de maquina listo para ser ejecutado por el sistema operativo. Los errores en esta etapa son menos comunes, pero pueden incluir problemas de permisos o errores de formato del archivo ejecutable.

9 )¿Qué ventaja ofrece un lock raii frente al tradicional lock/unlock?
Un lock RAII (Resource Acquisition Is Initialization) ofrece varias ventajas frente al tradicional lock/unlock:

1. Seguridad: El lock RAII asegura que el recurso se libere automáticamente cuando el objeto
2. sale de su alcance, evitando así posibles fugas de recursos o deadlocks.
3. Simplicidad: El uso de lock RAII simplifica el código, ya que no es necesario recordar liberar el recurso manualmente, lo que reduce la posibilidad de errores.
4. Mantenimiento: El código es más fácil de mantener y entender, ya que la gestión de recursos está encapsulada en un objeto, lo que hace que el flujo de control sea más claro y menos propenso a errores.

10)¿Qué significa que una función es blocante?¿Cómo subsanaría esa limitación en términos de mantener el programa ‘vivo’?

Una funcion es bloqueante cuando su ejecucion detiene todo al resto del programa sin permitir que otras operaciones se realicen hasta que termine. Esto puede causar que la aplicacion se congele o no responda. Se podria resolver esta limitacion utilizando hilos para ejecutar la funcion bloqueante en un hilo separado permitiendo al resto continuar el programa.

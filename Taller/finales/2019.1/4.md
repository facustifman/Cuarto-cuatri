1) ¿Qué es una macro de C? Enumere buenas prácticas para su definición.

Una macro de C es una directiva del procesador que permite definir fragmentos de código que pueden ser reutilizados en diferentes partes del programa. Se definen utilizando la palabra clave `#define` y pueden ser utilizadas para crear constantes, funciones en línea o para realizar operaciones condicionales.
Buenas prácticas para la definición de macros incluyen:

- Usar nombres en mayúsculas para las macros, para diferenciarlas de las variables y funciones.
- Evitar el uso de macros para funciones complejas; en su lugar, utilizar funciones en línea o plantillas.
- Documentar claramente el propósito de la macro. 

2) Describa el proceso de transformación de código fuente a un ejecutable. Precise las etapas y
las tareas desarrolladas en cada una de ellas.

1. Preprocesamiento: En esta etapa, el procesador de C/C++ realiza todas las directivas de preprocesamiento, como la inclusion de archivos de encabezado, la expansion de macros y la eliminacion de comentarios. Los errores comunes en esta etapa incluyen errores de sintaxis en las directivas de preprocesamiento y problemas con la inclusión de archivos.
2. Compilacion: El compilador traduce el codigo fuente a codigo objeto, que es un formato intermedio. Durante esta etapa, se realizan verificaciones de sintaxis y semántica. Los errores comunes incluyen errores de sintaxis, tipos de datos incompatibles y referencias a funciones o variables no definidas.
3. Enlazado: El linker combina los archivos de codigo objeto y las biblotecas necesarias para crear un ejecutable. En esta etapa, se resuelven las referencias a funciones y variables externas. Los errores comunes que incluye es que las referencias a funciones o variables no definidas, conflictos de nombres y problemas con las bibliotecas compartidas.
4. Generacion del ejecutable :  En esta etapa final, el linker genera el archivo ejecutable final con el codigo de maquina listo para ser ejecutado por el sistema operativo. Los errores en esta etapa son menos comunes, pero pueden incluir problemas de permisos o errores de formato del archivo ejecutable.

1) Escriba un programa (desde la inicialización hasta la liberación de los recursos) que reciba
paquetes de la forma nnn+nn+….+nnnn= (numeros separados por +, seguidos de =) e
imprima el resultado de la suma de cada paquete por pantalla. Al recibir un paquete vacío
(“=”) debe cerrarse ordenadamente. No considere errores.

```C++
#include <iostream>
#include <cctype>
#include <unistd.h>
#include <sys/socket.h>
#include <arpa/inet.h>

int recvall(int fd, char* buffer, int len) {
    int recibidos = 0;
    while (recibidos < len) {
        int n = recv(fd, buffer + recibidos, len - recibidos, 0);
        if (n <= 0) {
            return n;
        }
        recibidos += n;
    }
    return recibidos;
}

int main() {
    int server_fd = socket(AF_INET, SOCK_STREAM, 0);

    sockaddr_in addr{};
    addr.sin_family = AF_INET;
    addr.sin_addr.s_addr = INADDR_ANY;
    addr.sin_port = htons(5000);

    bind(server_fd, (struct sockaddr*)&addr, sizeof(addr));
    listen(server_fd, 1);              // una unica conexion

    int client_fd = accept(server_fd, nullptr, nullptr);

    int numero = 0;
    int suma = 0;
    bool huboNumero = false;   // si el paquete actual ya leyo algun digito

    char c;
    while (recvall(client_fd, &c, 1) > 0) {
        if (isdigit(c)) {
            numero = numero * 10 + (c - '0');
            huboNumero = true;
        } else if (c == '+') {
            suma += numero;
            numero = 0;
        } else if (c == '=') {
            if (!huboNumero) {          // paquete vacio: "="
                close(client_fd);
                close(server_fd);
                return 0;
            }
            suma += numero;
            std::cout << suma << std::endl;
            suma = 0;
            numero = 0;
            huboNumero = false;
        }
    }

    close(client_fd);
    close(server_fd);
    return 0;
}
```

2)Explique breve y concretamente qué es f: int (*f) (short*, char[4]);

La declaración `int (*f) (short *, char[4]);` define un puntero a función llamado `f`. Esta función apunta a una función que toma dos parámetros: un puntero a `short` y un arreglo de 4 caracteres (`char[4]`). La función devuelve un valor de tipo `int`. En otras palabras, `f` puede ser utilizado para llamar a cualquier función que tenga esa firma específica.

3)Analice el siguiente código y determine lo que se imprime (valor de Pi)
main()
{
int *Pi=1000;
Pi++;
printf(“Pi apunta a la dirección: %l”, (long)Pi);
}

El código tiene un error de tipo. La variable `Pi` se declara como un puntero a `int`, pero se le asigna el valor `1000`, que es un entero, no una dirección de memoria válida. Esto puede causar un comportamiento indefinido cuando se intenta incrementar el puntero con `Pi++`.  

Si asumimos que el código se ejecuta sin errores y que `Pi` apunta a una dirección de memoria válida, al incrementar `Pi` con `Pi++`, el puntero avanzará al siguiente entero en la memoria. En un sistema donde un `int` ocupa 4 bytes, esto significa que `Pi` apuntará a la dirección `1004`. Por lo tanto, la salida del programa sería:

```C++
Pi apunta a la dirección: 1004
```

4 )¿Qué es un functor? ¿Qué ventaja ofrece frente a una función convencional? Ejemplifique

Un functor, también conocido como objeto funcional, es un objeto de una clase que sobrecarga el operador de llamada a función `operator()`. Esto permite que el objeto se comporte como si fuera una función, es decir, se puede "llamar" al objeto como si fuera una función.

La ventaja de un functor frente a una función convencional es que un functor puede mantener estado interno entre llamadas, lo que permite almacenar información o configuraciones específicas para cada instancia del functor. Además, los functors pueden ser más flexibles y reutilizables, ya que pueden encapsular comportamientos complejos y ser utilizados en algoritmos genéricos.

Ejemplo de un functor:

```C++
#include <iostream>

class SumaFunctor {
private:
    int valor;
public:
    SumaFunctor(int v) : valor(v) {}
    int operator()(int x) {
        return valor + x;
    }
};

int main() {
    SumaFunctor f(5);
    std::cout << f(10) << std::endl;  // Imprime 15
    return 0;
}
```

5)¿Cómo se logra que 2 threads accedan (lectura/escritura) a un mismo recurso compartido
sin que se generen problemas de consistencia? Ejemplifique.

Para que dos threads accedan a un mismo recurso compartido sin generar problemas de consistencia, se utiliza el mecanismo de sincronizacion conocido como mutex. Un mutex es un objeto que permite que un solo thread acceda a un recurso compartido a la vez, garantizando que no haya condiciones de carrera ni inconsistencias en los datos.

Ejemplo de uso de mutex en C++:

```C++
#include <iostream>
#include <thread>
#include <mutex>

class SharedResource {
    private:
        int data;
        std::mutex mtx; // Mutex para proteger el acceso a 'data'
    public:
        SharedResource() : data(0) {}

        void increment() {
            std:: lock_guard<std::mutex> lock(mtx); // Bloquea el mutex
            data++;
        }
        void print() {
            std::lock_guard<std::mutex> lock(mtx); // Bloquea el mutex
            std::cout << "Data: " << data << std::endl;
        }
};

void threadFunction(SharedResource& resource) {
    for (int i = 0; i < 100; ++i) {
        resource.increment();
    }
}

int main() {
    SharedResource resource;

    std::thread t1(threadFunction, std::ref(resource));
    std::thread t2(threadFunction, std::ref(resource));

    t1.join();
    t2.join();

    resource.print(); // Imprime el valor final de 'data'

    return 0;
}
```

6)Describa el concepto de loop de eventos (events loop) utilizado en programación orientada
a eventos y, en particular, en entornos de interfaz gráfica (GUIs)

El loop de eventos (event loop) es un patron de programación utilizado en entornos de programación orientada a eventos, especialmente en aplicaciones con interfaces gráficas de usuario (GUIs). Su función principal es esperar y gestionar eventos generados por el usuario o por el sistema, como clics del ratón, pulsaciones de teclas, movimientos del ratón, entre otros.
El loop de eventos funciona de la siguiente manera:
1. Inicialización: Se configura la aplicación y se inicializan los componentes de la GUI.
2. Espera de eventos: El loop entra en un estado de espera, monitoreando continuamente la cola de eventos para detectar cualquier evento que ocurra.
3. Procesamiento de eventos: Cuando se detecta un evento, el loop lo procesa llamando a la función o método correspondiente que maneja ese evento específico. Esto puede incluir actualizar la interfaz de usuario, realizar cálculos, o ejecutar otras acciones según la lógica de la aplicación.
4. Repetición: Después de procesar el evento, el loop vuelve a esperar nuevos eventos, repitiendo el ciclo.
El loop de eventos permite que la aplicación responda de manera eficiente a las interacciones del usuario y a otros eventos del sistema, manteniendo la interfaz de usuario activa y receptiva. Es un componente esencial en la programación de aplicaciones GUI, ya que facilita la gestión de múltiples eventos de manera concurrente y organizada, evitando bloqueos y asegurando una experiencia de usuario fluida.

No, no es verdad que `sizeof(ejemplo) = sizeof(a) + sizeof(b)`. La razón es que la estructura puede tener un "padding" o relleno agregado por el compilador para alinear los miembros de la estructura en memoria. Esto se hace para optimizar el acceso a los datos y cumplir con las restricciones de alineación del hardware.

8)¿En qué consiste el patrón de diseño RAII? Ejemplifique.

RAII es un patrón de diseño en C++ que significa "Resource Acquisition Is Initialization" (Adquisición de Recursos es Inicialización). La idea principal es que la adquisición de un recurso (memoria, archivos, sockets, mutex, etc.) se realiza en el constructor del objeto que lo administra, y su liberación se realiza en el destructor. Como en C++ el destructor de un objeto local se ejecuta automáticamente al salir del scope —ya sea por un `return` normal, un `break`, o porque se propaga una excepción—, el recurso queda liberado sin necesidad de llamar manualmente a una función de liberación, y sin riesgo de fugas incluso ante errores.

Para que la clase sea realmente segura también hay que resolver el problema de la copia: si se deja que el compilador genere el constructor de copia y el operador de asignación por defecto, dos objetos podrían terminar administrando el mismo recurso, y al destruirse ambos se liberaría dos veces (double free / double close). Por eso una clase RAII correcta prohíbe la copia (o la implementa explícitamente, por ejemplo con move semantics).

Ejemplo de RAII:

```C++
#include <iostream>
#include <cstdio>
#include <stdexcept>

class FileHandler {
private:
    FILE* file;
public:
    FileHandler(const char* filename) {
        file = fopen(filename, "r");
        if (!file) {
            throw std::runtime_error("No se pudo abrir el archivo");
        }
    }

    ~FileHandler() {
        if (file) {
            fclose(file);
        }
    }

    // se prohibe la copia: dos objetos no pueden administrar el mismo FILE*
    FileHandler(const FileHandler&) = delete;
    FileHandler& operator=(const FileHandler&) = delete;

    void read() {
        // Leer del archivo
    }
};

int main() {
    try {
        FileHandler fh("datos.txt");
        fh.read();
        // no hace falta llamar a fclose(): se libera solo al salir del scope,
        // incluso si read() lanzara una excepcion
    } catch (const std::runtime_error& e) {
        std::cerr << e.what() << std::endl;
    }

    return 0;
}
```

9)Escribir un programa ISO C que procese el archivo de enteros de 2 bytes bigendian cuyo
nombre es recibido como parámetro. El procesamiento consiste en eliminar los número
múltiplos de 3, trabajando sobre el mismo archivo (sin archivos intermedios ni en
memoria).

```C
#include <stdio.h>
#include <unistd.h>

short int swap16(short int val) {
    return (short int)(((val & 0xFF) << 8) | ((unsigned short)val >> 8));
}

int main(int argc, char *argv[]) {
    if (argc != 2) {
        return 1;
    }

    FILE *f = fopen(argv[1], "rb+");
    if (f == NULL) {
        return 1;
    }

    long readPos = 0;
    long writePos = 0;
    short int num;

    while (1) {
        fseek(f, readPos, SEEK_SET);
        if (fread(&num, sizeof(short int), 1, f) != 1) {
            break;
        }
        readPos += sizeof(short int);

        if (swap16(num) % 3 != 0) {
            fseek(f, writePos, SEEK_SET);
            fwrite(&num, sizeof(short int), 1, f);
            writePos += sizeof(short int);
        }
    }

    fflush(f);
    ftruncate(fileno(f), writePos);
    fclose(f);
    return 0;
}

10)Implemente una función C++ denominada DobleSiNo que reciba dos listas de elementos y
devuelva una nueva lista duplicando los elementos de la primera que no están en la
segunda:
std::list<T> DobleSiNo(std::list<T> a,std::list<T> b);

```C++
#include <list>
template <typename T>
std::list<T> DobleSiNo(std::list<T> a, std::list<T> b) {
    std::list<T> result;
    for (const auto& item : a) {
        if (std::find(b.begin(), b.end(), item) == b.end()) {
            result.push_back(item * 2);
        }
    }
    return result;
}
```

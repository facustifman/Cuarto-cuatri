Una variable global static es una variable que se declara fuera de cualquier función y tiene un alcance limitado al archivo en el que se declara. Su inicialización ocurre una sola vez, y su valor persiste durante toda la ejecución del programa. Reside en el área de memoria de datos estáticos.

una variable local static es una variable que se declara dentro de una función y tiene un alcance limitado a esa función. Su inicialización ocurre una sola vez, y su valor persiste entre llamadas a la función. Reside en el área de memoria de datos estáticos.

Un atributo de clase static es una variable que pertenece a la clase en lugar de a una instancia específica de la clase. Su inicialización ocurre una sola vez, y su valor persiste durante toda la ejecución del programa. Reside en el área de memoria de datos estáticos.

3)Escribir un programa ISO C que procese el archivo “nros2bytes.dat” sobre sí mismo,
duplicando los enteros de 2 bytes múltiplos de 3.

``` C++ 

#include <stdio.h>

int main(void) {
    FILE *f = fopen("nros2bytes.dat", "rb+");
    if (f == NULL) {
        return 1;
    }

    short int num;
    long pos;

    while (fread(&num, sizeof(short int), 1, f) == 1) {
        if (num % 3 == 0) {
            num *= 2;
            pos = ftell(f);                        /* posición tras leer */
            fseek(f, pos - (long)sizeof(short int), SEEK_SET);
            fwrite(&num, sizeof(short int), 1, f);
            fseek(f, pos, SEEK_SET);                /* volver a donde estaba */
        }
    }

    fclose(f);
    return 0;
}
```

4)¿Cómo se logra que 2 threads accedan (lectura/escritura) a un mismo recurso compartido
sin que se generen problemas de consistencia? Ejemplifique.

Para que dos threads accedan a un mismo recurso compartido sin problemas de consistencia, se utilizan mecanismos de sincronización como mutexes (mutual exclusion locks). Un mutex asegura que solo un thread pueda acceder al recurso compartido a la vez, evitando condiciones de carrera.

```C++
#include <iostream>
#include <thread>
#include <mutex>

class SharedResource {
private:
    int data = 0;
    std::mutex mtx; // protege el acceso a 'data'

public:
    void write(int value) {
        std::lock_guard<std::mutex> lock(mtx); // toma el mutex
        data = value;
        // al salir del scope, lock_guard libera el mutex automáticamente
    }

    int read() {
        std::lock_guard<std::mutex> lock(mtx);
        return data;
    }
};

void writer(SharedResource& res) {
    for (int i = 0; i < 100; ++i) {
        res.write(i);
    }
}

void reader(SharedResource& res) {
    for (int i = 0; i < 100; ++i) {
        int value = res.read();
        std::cout << "Leido: " << value << std::endl;
    }
}

int main() {
    SharedResource res;

    std::thread t1(writer, std::ref(res));
    std::thread t2(reader, std::ref(res));

    t1.join();
    t2.join();

    return 0;
}


```

6)¿Qué es una macro de C? Detalle las buenas prácticas para su definición. Ejemplifique

Una macro de C es una directiva del procesador que define un nombre simbólico para un valor o una expresión. Se utiliza para reemplazar texto en el código fuente antes de la compilación. Las macros se definen utilizando la directiva `#define`.

Buenas prácticas para la definición de macros incluyen:

- Usar nombres en mayúsculas para distinguirlas de variables y funciones.
- Evitar el uso de macros para funciones complejas; en su lugar, usar funciones inline
- Evitar efectos secundarios en las macros, ya que pueden llevar a resultados inesperados.
- Ejemplo de macro:

```C
#define PI 3.14159
#define SQUARE(x) ((x) * (x)) // Buena práctica: usar paréntesis para evitar problemas de precedencia
```

7)Describa el proceso de transformación de código fuente a un ejecutable. Precise las etapas y las tareas desarrolladas en cada una de ellas.

El proceso de transformación de código fuente a un ejecutable consta de varias etapas:

1. Preprocesamiento: El preprocesador realiza tareas como la inclusión de archivos de encabezado, la expansión de macros y la eliminación de comentarios. El resultado es un archivo de código fuente expandido.
2. Compilación: El compilador traduce el código fuente expandido a código objeto, que es un lenguaje de bajo nivel (generalmente ensamblador o código máquina). Esta etapa también realiza optimizaciones y verifica la sintaxis del código.
3. Ensamblado: El ensamblador convierte el código objeto en un archivo de código máquina que puede ser entendido por la CPU. El resultado es un archivo de objeto.
4. Enlazado: El enlazador combina uno o más archivos de objeto y bibliotecas para crear un ejecutable final. Resuelve referencias a funciones y variables externas, y organiza el código y los datos en la memoria. El resultado es un archivo ejecutable que puede ser cargado y ejecutado por el sistema operativo.
5. Carga y ejecución: El sistema operativo carga el ejecutable en la memoria y comienza su ejecución, iniciando el flujo del programa desde la función principal.
6. Finalización: El programa termina su ejecución y el sistema operativo libera los recursos utilizados.

9) Implemente una función C++ denominada Sacar que reciba dos listas de elementos y
devuelva una nueva lista con los elementos de la primera que no están en la segunda:
std::list<T> Sacar(std::list<T> a,std::list<T> b);

```C++
template <typename T>
std::list<T> Sacar(std::list<T> a, std::list<T> b) {
    std::list<T> result;
    for (const auto& item : a) {
        if (std::find(b.begin(), b.end(), item) == b.end()) { //sii llego al final y el item no esta en b, lo agrego a result
            result.push_back(item);
        }
    }
    return result;
}
```

10)Escriba un programa que reciba por línea de comandos un Puerto y una IP. El programa debe establecer una unica conexión, quedar en escucha e imprimir en stdout todo lo recibido. Al recibir el texto ‘FINAL’ debe finalizar el programa sin imprimir dicho texto.

```C++
#include <iostream>
#include <string>
#include <unistd.h>
#include <sys/socket.h>
#include <arpa/inet.h>

int main(int argc, char *argv[]) {
    const char *ip = argv[1];
    int puerto = atoi(argv[2]);

    int server_fd = socket(AF_INET, SOCK_STREAM, 0);

    sockaddr_in addr{};
    addr.sin_family = AF_INET;
    addr.sin_port = htons(puerto);
    inet_pton(AF_INET, ip, &addr.sin_addr);

    bind(server_fd, (struct sockaddr*)&addr, sizeof(addr));
    listen(server_fd, 1);              // 1 = una unica conexion

    int client_fd = accept(server_fd, nullptr, nullptr);

    std::string acumulado;
    char buffer[1024];
    int n;

    while ((n = recv(client_fd, buffer, sizeof(buffer), 0)) > 0) {
        acumulado.append(buffer, n);
        size_t pos = acumulado.find("FINAL");
        if (pos != std::string::npos) {
            std::cout << acumulado.substr(0, pos);
            break;                      // no imprime "FINAL"
        }
        size_t seguro = acumulado.size() > 4 ? acumulado.size() - 4 : 0;
        std::cout << acumulado.substr(0, seguro);
        acumulado.erase(0, seguro);
    }

    close(client_fd);
    close(server_fd);
    return 0;
}

```
